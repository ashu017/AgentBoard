## Affected Pages/Routes
- `/` (board) — default after login; live Kanban, four columns, scan summary, states.
- `/agents` — agent list, create-agent + shown-once key, connected indicator, revoke.
- `/tasks/new` (or modal) — create + assign task to one agent.
- `/api/mcp` — MCP server (stateless Streamable-HTTP); agent tools.
- GitHub OAuth login + first-login workspace bootstrap.

## Key Interactions to Verify
- Create agent → key shown exactly once → copy → "I've saved it" dismiss (irreversible).
- Connected indicator flips grey→green on the agent's first MCP call.
- Create + assign task → card appears on board live (Realtime).
- Agent updates status via MCP → card moves column live.
- Revoke agent → next MCP call returns 401.

## Edge Cases
- New user first login: workspace auto-created, board not broken/empty-crash.
- No agents → board shows "Add your first agent" CTA, not a dead grid.
- No active agents → task creation blocked, routed to add-agent.
- Double-click submit on create task/agent → locked, no duplicate.
- Realtime disconnect → "reconnecting…" then refetch on reconnect.
- Oversize result (>256KB) → 413.
- Illegal status transition (out of terminal) → 409.
- Revoked/unknown key → 401.

## Critical Paths
- Cross-tenant isolation: agent A cannot read/write workspace B (data-breach guard).
- Human-plane RLS: user A cannot see user B's data.
- Full loop: login → create agent → assign task → agent reports via MCP → board updates.
- Real MCP client connects to the stateless endpoint (agent-plane gate).
